## 2. 验收范围与原则

1. 功能验收以附件1中带编号的叶子功能点（【Rxx-xxx】）为准。
2. 未写入附件1/本附件的内容不作为验收标准；新增需求按合同变更流程处理。
3. 验收环境默认：乙方测试环境/甲方验收环境/甲方生产环境（上线阶段），由甲方提供必要账号与网络条件。

## 3. 验收流程（与合同衔接）

1. 乙方提交《验收申请》与版本说明（含部署地址、账号、变更清单、已知问题）。
2. 甲方在**5个工作日**内一次性给出《缺陷清单/不通过原因》（需可复现、可定位、与验收项对应）。
3. 乙方修复后提交复验；复验期**5个工作日**。

## 4. 缺陷分级与通过规则

| 等级 | 定义 | 是否可导致不通过 |
|---|---|---|
| P0 致命 | 系统无法启动/核心流程不可用/数据严重错误且可复现/跨校越权 | 是 |
| P1 严重 | 核心流程可用但存在明显错误/安全风险/数据不一致 | 视情况（默认需修复） |
| P2 一般 | 非核心问题、体验问题、文案样式 | 否（进入质保期修复或变更） |

> 一票否决项：任何“未经授权跨校查询/修改数据”的缺陷，直接判定为不合格交付。

## 5. 性能与并发验收标准（可量化）

### 5.1 响应时间（P95）
- 简单查询接口（单表/无复杂计算）：≤ 200ms。
- 复杂报表/聚合统计：≤ 1000ms。

### 5.2 并发能力
- 单服务常规业务接口：支持 ≥ 1000 QPS（在约定压测环境与数据规模下）。

### 5.3 约束与方法
- 列表接口必须分页，`limit` 最大值 ≤ 100。
- 复杂报表优先采用：异步任务（Asynq）+ 结果缓存（Redis，TTL 5~30 分钟）或物化视图/异步聚合。
- 必须开启 Jaeger 链路追踪，用于定位慢查询与跨服务瓶颈。

## 6. 测试与质量门禁

### 6.1 单元测试
- 核心业务逻辑（计费/排课算法/权限计算等）单元测试覆盖率 **≥80%**。

### 6.2 集成测试
- 至少覆盖：登录鉴权 → 权限校验 → 关键业务写入 → 通知触达 → 审计日志落库/落盘 的端到端链路。

### 6.3 代码质量与安全（作为验收门禁）
- `golangci-lint run ./...` 无阻断级问题。
- 代码已执行 `gofmt`/`goimports`。
- 禁止硬编码敏感信息（DB/Redis/JWT Secret/第三方 Token）。
- UUID 入参必须 `uuid.Parse()` 校验后再进入数据查询。
- 不得存在 SQL 注入、XSS 等安全漏洞。

## 7. 功能验收模板（用于逐条判定）

> 下表中的模板编号将被“功能验收映射表”引用。所有模板均默认叠加通用门禁：租户隔离（school_id）、权限校验（RBAC）、审计/日志（如适用）、错误码与输入校验。

### T-CRUD：增删改查（含列表）
- 通过标准：完成新增/编辑/删除（软删）/详情或列表查询；列表分页、`limit<=100`；数据严格按 `school_id` 隔离；操作有审计（管理端）。

### T-LIST：列表/查询
- 通过标准：支持分页、筛选（如关键字/状态）、排序（如有）；返回结构与 API 文档一致；跨校越权不可查。

### T-CREATE：新增/登记/申请/绑定
- 通过标准：字段校验明确；创建成功返回 UUID；`school_id` 正确写入且不可修改；重复/非法输入有明确错误提示。

### T-UPDATE：编辑/审批/驳回/设置
- 通过标准：更新仅允许修改允许字段；`school_id` 不可变；审计记录完整；并发更新有冲突处理（如乐观锁/更新时间校验）。

### T-DELETE：删除/撤消/屏蔽
- 通过标准：默认软删除；删除后列表不可见、详情不可访问（或标记已删除）；具备恢复策略（如合同另约）。

### T-AUTH：注册/登录/SSO/找回
- 通过标准：支持指定登录方式；成功后返回 JWT 且包含 `org_id/org_type/role`；权限与租户上下文可解析。

### T-NOTIFY：短信/微信/站内通知/报警
- 通过标准：模板/通道可配置；发送记录可追踪状态；支持延迟/撤回（如需求要求）；失败可重试且有告警。

### T-IMPORT：批量导入
- 通过标准：支持模板校验/错误行回显；大文件采用异步任务；导入结果可查询；不跨校写入。

### T-EXPORT：导出/打印
- 通过标准：导出数据与查询条件一致；权限与脱敏规则生效；文件可追溯生成时间与操作者。

### T-SCHEDULE：排课/课程/作息/冲突规则
- 通过标准：关键约束生效（教师/班级/教室冲突等）；可按条件预排/调整；结果可视化展示并可回溯。

### T-REPORT：统计/报表/大屏
- 通过标准：指标口径一致；复杂查询满足性能指标；汇总视角遵循 `X-View-Scope` 与授权子树。

### T-GENERAL：通用功能点
- 通过标准：按需求描述实现并可演示/可复现；接口/页面有明确输入输出与错误处理。
