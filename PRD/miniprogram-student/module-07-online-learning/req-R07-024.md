# 【R07-024】我的课程
<a id="R07-024"></a>

- **Requirement ID**：【R07-024】
- **Feature name**：我的课程
- **Source**：模块07：在线学习 15% > 课程跟踪 > 我的课程
- **Applicable Platform/Entry**：小程序（学生端） / 模块07 / 我的课程
- **User Roles & Permission Requirements**：
  - 可执行角色：学生
  - 权限点（最小粒度）：`req:R07-024`
  -（可选）通用权限点：`learning:schedule`
  - 视角/范围：不涉及后台双视角切换；以绑定/所属学校 `school_id` 与身份关系为边界。
- **User Stories**：
  - 场景 1：作为学生，我希望能够完成“我的课程”，以便在权限允许且 `school_id` 隔离范围内推进业务。
  - 场景 2：作为学生，我希望设置关键约束并生成/调整排课结果，系统能避免教师/班级/教室冲突。
  - 场景 3：作为移动端用户，我希望在弱网/重复点击时系统能避免重复提交并给出明确提示。
- **Business Rules & Boundaries**：
  - 输入校验：所有 UUID 入参必须校验格式；非法输入应返回可定位的错误。
  - 关键排课约束需生效（教师/班级/教室冲突等），且冲突在同一 `school_id` 范围内判定。
  - 排课结果应可视化展示，并可回溯生成条件/调整记录（最小可验收）。
- **Data Scope（school_id）**：
  - 仅允许访问与当前用户身份绑定的 `school_id` 数据（不可跨校）。
  - 仅允许访问与本人/授权关系相关的数据（学生本人、监护关系、任教/岗位范围等）。
- **Audit Log Requirements**：
  - Recommended：登录/绑定/信息修改/提交等关键行为建议记录操作日志（至少包含 operator_id、school_id、action、resource_type/id、timestamp）。
- **Acceptance Criteria（verifiable）**：
  - 验收模板：`T-SCHEDULE`
  - 验收用例：
    1. 权限校验：具备 `req:R07-024`（及必要通用权限点）才允许执行；无权限必须拒绝。
    2. 租户隔离：尝试跨 school_id（或未授权 X-View-Scope）访问/操作必须失败；不得出现跨校可查/可改。
    3. 输入校验：UUID/必填字段/格式校验失败时应返回明确错误。
    4. 排课约束：教师/班级/教室等关键冲突约束必须生效；冲突需可复现并可定位。
    5. 排课调整：支持按条件预排/调整；调整结果可视化展示且可回溯调整记录。
